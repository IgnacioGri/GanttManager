Creá un sitio web simple para la creación y gestión de diagramas de Gantt. Empezá con funcionalidades básicas y mantenelo minimalista para que sea fácil de iterar. Usá tecnologías estándar como HTML, CSS y JavaScript para el frontend, y Node.js con Express para el backend si es necesario para el almacenamiento y manejo de archivos. Para el diagrama de Gantt, integrá una librería open-source como D3.js o una más específica como Frappe Gantt.
Funcionalidades iniciales obligatorias:
	•	Creación de Gantt: Permití agregar tareas con nombre, fecha de inicio, fecha de fin, duración, progreso, dependencias entre tareas, y opción para adjuntar un documento (como PDF o DOCX para casos de uso). Visualizá el timeline horizontal con barras para tareas, líneas para dependencias y escalas de tiempo ajustables (días, semanas, meses). Para cada tarea, incluí un botón que abra un pop-up con un block de notas editable titulado “Comentarios”, persistiendo el contenido en el almacenamiento.
	•	Guardado y creación nueva: Botón para crear un nuevo Gantt vacío. Guardá el Gantt actual en almacenamiento local o en un servidor simple (usá JSON para serializar los datos, incluyendo adjuntos como base64 o rutas de archivos y comentarios como texto).
	•	Abrir, visualizar y editar: Listá los Gantts guardados en una interfaz inicial. Permití seleccionar uno para cargarlo, visualizarlo y editarlo directamente en la vista principal, manteniendo acceso a adjuntos y comentarios por tarea.
Funcionalidad adicional prioritaria:
	•	Exportación a Excel: Generá un archivo XLSX legible y estética, con columnas para tarea, inicio, fin, duración, progreso, dependencias, enlace o referencia al documento adjunto, y extracto o enlace a comentarios. Usá una librería como SheetJS para formatear el output con colores, bordes y alineaciones que reflejen la estructura del Gantt.
Asegurate de que la interfaz sea intuitiva, responsive y sin dependencias innecesarias. Proporcioná el código completo, estructurado en archivos, y explicá brevemente cómo deployarlo localmente. Una vez listo, sugerí mejoras como autenticación o integración con bases de datos para versiones futuras.
